# Contributing Guide

Thanks for your interest in improving this portfolio repository.

## Workflow

1. Fork the repository
2. Create a feature branch
3. Make focused changes
4. Open a pull request with a clear summary

## Standards

- Keep edits scoped and minimal
- Preserve existing code style and formatting
- Test static rendering locally before submitting
- Do not add secrets or credentials

## Commit Messages

Use clear, action-oriented messages.

Examples:

- `Improve SEO metadata and sitemap`
- `Harden JS selectors to prevent runtime errors`
- `Fix malformed HTML around testimonials`

## Content Changes

When editing copy in `index.html`:

- Keep claims realistic and verifiable
- Prefer measurable outcomes over generic statements
- Keep tone professional and concise
